# ChatGPT para la Oficina — Versión Gratuita

## 5 plantillas prácticas para ahorrar tiempo en tareas laborales

---

Esta versión gratuita reúne cinco plantillas prácticas para usar ChatGPT en tareas comunes de oficina: redactar correos, responder mensajes difíciles, resumir reuniones, extraer compromisos y preparar presentaciones.

La idea es que puedas probar una forma simple de pedir mejores respuestas, ahorrar tiempo y mantener el control sobre el resultado final.

---

# Cómo usar estas plantillas

Cada plantilla incluye:

- para qué sirve
- cuándo usarla
- prompt listo para copiar
- ejemplo de entrada
- ejemplo de salida esperada
- consejo práctico
- advertencia de uso

Copia el prompt, reemplaza los campos entre corchetes con tu propia información y pega el resultado en ChatGPT.

Antes de usar cualquier respuesta generada con IA, revisa que el contenido sea correcto, que el tono calce con tu situación y que no incluya información inventada.

---

# Plantillas incluidas

A continuación encontrarás cinco plantillas seleccionadas de la versión completa.



# Plantilla 1: Mejorar un correo mal redactado

## Para qué sirve

Sirve para transformar un correo desordenado, muy largo, poco claro o mal redactado en un mensaje profesional, entendible y fácil de responder.

La idea no es que ChatGPT invente el mensaje, sino que te ayude a ordenar mejor lo que ya quieres decir.

---

## Cuándo usarla

Úsala cuando tienes que enviar un correo laboral y sientes que:

- escribiste muchas ideas sueltas
- el tono quedó muy informal
- el mensaje puede sonar brusco
- no sabes cómo ordenar la información
- quieres sonar más claro, profesional y humano
- necesitas una versión más breve y directa

Ejemplos típicos:

- correo a un jefe
- correo a un cliente
- correo a un proveedor
- respuesta a un compañero de trabajo
- seguimiento de una tarea pendiente
- solicitud de información

---

## Prompt listo para copiar

```text
Actúa como un asistente de redacción profesional.

Necesito mejorar el siguiente correo para que sea más claro, ordenado y profesional, pero sin sonar robótico ni exageradamente formal.

Correo original:
[pega aquí tu correo]

Contexto:
[explica brevemente la situación]

Destinatario:
[indica si es para un jefe, cliente, proveedor, compañero, equipo interno, etc.]

Objetivo del correo:
[explica qué quieres lograr con el correo]

Tono deseado:
[cordial / formal / directo / cercano / firme / colaborativo]

Restricciones:
[indica si hay algo que no quieres decir, prometer o mencionar]

Devuélveme:
1. Una versión mejorada del correo.
2. Una versión más breve y directa.
3. Tres recomendaciones para mejorar el mensaje.
```

## Ejemplo de entrada

Correo original:
Hola Juan, te escribo porque todavía no me mandan los datos que pedimos la semana pasada y estamos atrasados con el informe. Necesito que nos puedan ayudar con eso porque si no vamos a quedar parados. Igual entiendo que están con hartas cosas, pero necesitamos avanzar.

Contexto:
Estamos esperando información de otra área para cerrar un informe mensual.

Destinatario:
Compañero de otra área.

Objetivo del correo:
Solicitar el envío de la información pendiente y dejar claro que es necesaria para avanzar.

Tono deseado:
Cordial, claro y directo.

Restricciones:
No quiero sonar molesto ni acusador.

## Ejemplo de salida esperada

Hola Juan,

Espero que estés bien.

Te escribo para hacer seguimiento a la información que solicitamos la semana pasada, ya que la necesitamos para poder avanzar con el cierre del informe mensual.

Entiendo que pueden estar con varias prioridades en paralelo, pero nos ayudaría mucho contar con esos datos durante el día, o al menos tener una fecha estimada de envío.

Quedo atento. Muchas gracias por el apoyo.

Saludos,
[Tu nombre]

## Versión más breve

Hola Juan,

Te escribo para hacer seguimiento a la información pendiente que solicitamos la semana pasada. La necesitamos para avanzar con el cierre del informe mensual.

¿Nos podrías confirmar si es posible enviarla durante el día o indicarnos una fecha estimada?

Gracias por el apoyo.

Saludos,
[Tu nombre]

## Consejo práctico

Mientras más contexto le des a ChatGPT, mejor será el resultado.

No le pidas simplemente:

- Mejora este correo.

Mejor dile:

- Mejora este correo para enviarlo a un compañero de otra área. Quiero sonar cordial, pero necesito dejar claro que estamos atrasados por falta de información.

La diferencia está en el contexto.

## Cuidado con

No uses la respuesta sin revisarla.
Antes de enviar el correo, verifica:

- [ ] ¿El mensaje dice exactamente lo que quiero decir?
- [ ] ¿El tono calza con la relación que tengo con la persona?
- [ ] ¿No promete algo que no puedo cumplir?
- [ ] ¿No incluye información inventada?
- [ ] ¿No suena demasiado artificial?

# Plantilla 2: Responder un correo difícil sin sonar agresivo

## Para qué sirve

Sirve para responder correos laborales incómodos, tensos o delicados sin sonar agresivo, defensivo o poco profesional.

La idea es ayudarte a mantener una postura clara, pero con un tono respetuoso y controlado.

Esta plantilla es útil cuando necesitas responder con firmeza, aclarar una situación, corregir una interpretación o pedir responsabilidad, pero sin escalar el conflicto.

---

## Cuándo usarla

Úsala cuando recibes un correo que:

- te parece injusto
- viene con reclamos o presión
- interpreta mal una situación
- te culpa por algo que no depende de ti
- exige una respuesta rápida
- tiene un tono molesto o poco colaborativo
- requiere aclarar responsabilidades

Ejemplos típicos:

- respuesta a un cliente molesto
- respuesta a una jefatura exigente
- respuesta a un proveedor que no cumplió
- respuesta a un compañero que trasladó mal una responsabilidad
- aclaración de un malentendido
- seguimiento de un compromiso pendiente

---

## Prompt listo para copiar

```text
Actúa como un asistente de redacción profesional.

Necesito responder el siguiente correo de forma clara, respetuosa y firme, sin sonar agresivo ni defensivo.

Correo recibido:
[pega aquí el correo recibido]

Contexto:
[explica brevemente qué pasó o qué información falta]

Mi postura:
[explica qué quieres dejar claro]

Objetivo de la respuesta:
[explica qué quieres lograr con la respuesta]

Relación con el destinatario:
[jefe / cliente / proveedor / compañero / equipo interno / otra área]

Tono deseado:
[cordial / firme / diplomático / directo / colaborativo]

Restricciones:
[indica qué no quieres decir, qué no puedes prometer o qué temas debes evitar]

Devuélveme:
1. Una respuesta recomendada.
2. Una versión más breve.
3. Una versión más firme, pero profesional.
4. Tres riesgos de comunicación que debería evitar.
```

## Ejemplo de entrada

Correo recibido:
Hola Diego, nuevamente estamos esperando respuesta de tu parte. Este atraso nos está afectando y necesitamos que el tema se resuelva hoy. No puede ser que cada vez que pedimos algo tengamos que insistir tanto.

Contexto:
La solicitud depende de información que todavía no nos ha entregado otra área. Yo ya hice seguimiento, pero aún no tengo respuesta.

Mi postura:
Quiero aclarar que no he dejado el tema sin gestión, pero tampoco quiero sonar confrontacional.

Objetivo de la respuesta:
Explicar el estado real, mostrar que estoy haciendo seguimiento y comprometerme a avisar apenas tenga novedades.

Relación con el destinatario:
Compañero de otra área.

Tono deseado:
Cordial, claro y firme.

Restricciones:
No quiero culpar directamente a la otra área ni generar conflicto.

## Ejemplo de salida esperada

Hola [Nombre],

Entiendo la urgencia y comparto la preocupación por avanzar con este tema.

Quiero aclararte que la solicitud no ha quedado sin gestión. Ya realicé el seguimiento correspondiente y, por ahora, seguimos a la espera de la información necesaria para poder cerrar la respuesta.

Apenas tenga novedades, te las compartiré de inmediato para que podamos avanzar lo antes posible.

De todas formas, haré un nuevo seguimiento durante el día para intentar acelerar la respuesta.

Saludos,
Diego

## Versión más breve

Hola [Nombre],

Entiendo la urgencia del tema.

Te confirmo que la solicitud ya fue gestionada y actualmente estamos a la espera de información necesaria para poder cerrar la respuesta.

Haré un nuevo seguimiento durante el día y te avisaré apenas tenga novedades.

Saludos,
Diego

## Versión más firme, pero profesional

Hola [Nombre],

Entiendo la urgencia y la necesidad de resolver este punto cuanto antes.

De todas formas, es importante aclarar que la solicitud ya fue gestionada y que el avance depende de información que aún está pendiente. Por mi parte, he realizado el seguimiento correspondiente y volveré a insistir durante el día.

Apenas tenga una actualización concreta, la compartiré de inmediato.

Saludos,
Diego

## Consejo práctico

Cuando un correo te molesta, no respondas de inmediato.

Primero escribe lo que realmente quisieras decir, sin filtro. Luego usa ChatGPT para transformarlo en una respuesta profesional.

Ejemplo de instrucción útil:

- Transforma esta respuesta en un correo profesional. Mantén mi postura, pero elimina cualquier tono molesto, irónico o defensivo.

Esto te ayuda a conservar el fondo del mensaje sin enviar una respuesta emocional.

## Cuidado con

No uses ChatGPT para “ganar la discusión”.

Úsalo para ordenar tu postura, bajar el conflicto y comunicar con más claridad.

Antes de enviar la respuesta, revisa:

- [ ] ¿Estoy respondiendo el punto principal?
- [ ] ¿El correo baja el conflicto o lo aumenta?
- [ ] ¿Estoy acusando a alguien innecesariamente?
- [ ] ¿Estoy prometiendo algo que no controlo?
- [ ] ¿La respuesta mantiene una postura clara?
- [ ] ¿El tono es profesional?

## Variante útil

También puedes pedirle a ChatGPT que evalúe el tono antes de enviar:

Evalúa el tono de este correo antes de enviarlo.

Correo:
[pega aquí tu respuesta]

Dime:

1. Si suena agresivo, defensivo o poco claro.
2. Qué parte podría generar conflicto.
3. Cómo mejorarlo sin perder firmeza.

# Plantilla 4: Resumir una reunión desde notas desordenadas

## Para qué sirve

Sirve para transformar notas desordenadas de una reunión en una minuta clara, ordenada y útil.

Esta plantilla ayuda a convertir apuntes rápidos, frases sueltas o transcripciones parciales en un resumen estructurado con acuerdos, pendientes y próximos pasos.

La idea no es que ChatGPT invente lo que ocurrió en la reunión, sino que ordene la información disponible.

---

## Cuándo usarla

Úsala cuando tienes:

- notas tomadas rápidamente durante una reunión
- una transcripción larga y poco clara
- apuntes mezclados con comentarios
- acuerdos repartidos en distintas partes del texto
- dudas sobre qué quedó pendiente
- necesidad de enviar una minuta al equipo

Ejemplos típicos:

- reunión de seguimiento
- reunión con cliente
- reunión interna de equipo
- reunión de proyecto
- revisión de avances
- levantamiento de requerimientos
- reunión comercial

---

## Prompt listo para copiar

```text
Actúa como un asistente de gestión y documentación de reuniones.

Necesito convertir estas notas desordenadas en una minuta clara, profesional y fácil de revisar.

Notas de la reunión:
[pega aquí tus notas, apuntes o transcripción]

Contexto de la reunión:
[explica brevemente de qué trataba la reunión]

Participantes:
[indica nombres o roles si los tienes]

Objetivo de la reunión:
[explica qué se buscaba resolver, revisar o decidir]

Formato requerido:
Minuta estructurada.

Instrucciones:
- No inventes información.
- Si falta un dato, escribe "No especificado".
- Separa claramente acuerdos, pendientes y próximos pasos.
- Usa lenguaje claro y profesional.

Devuélveme la minuta con esta estructura:

1. Tema de la reunión
2. Fecha
3. Participantes
4. Resumen general
5. Puntos tratados
6. Acuerdos
7. Pendientes
8. Responsables
9. Próximos pasos
10. Riesgos o alertas
```

## Ejemplo de entrada

Notas de la reunión:
Se revisó avance del informe mensual. Juan dice que todavía falta información del área comercial. María comenta que el diseño del reporte ya está casi listo. Se acuerda cerrar la primera versión el jueves si llegan los datos. Hay preocupación porque gerencia pidió revisar el informe el viernes temprano. Pedro queda de insistir con comercial. También se habló de mejorar el formato del resumen ejecutivo, pero eso puede quedar para la siguiente versión.

Contexto de la reunión:
Reunión interna para revisar el avance del informe mensual.

Participantes:
Juan, María, Pedro y equipo de análisis.

Objetivo de la reunión:
Revisar estado del informe, pendientes y fecha tentativa de cierre.

## Ejemplo de salida esperada

1. Tema de la reunión
   Revisión de avance del informe mensual.

2. Fecha
   No especificado.

3. Participantes
   Juan, María, Pedro y equipo de análisis.

4. Resumen general
   Se revisó el estado del informe mensual. El diseño del reporte está avanzado, pero aún falta información del área comercial para poder cerrar la primera versión. Existe preocupación por el plazo, ya que gerencia solicitó revisar el informe el viernes temprano.

5. Puntos tratados

- Avance general del informe mensual.
- Información pendiente del área comercial.
- Estado del diseño del reporte.
- Fecha tentativa de cierre.
- Revisión solicitada por gerencia.
- Mejora futura del resumen ejecutivo.

6. Acuerdos

- Cerrar la primera versión del informe el jueves, siempre que llegue la información pendiente.
- Dejar la mejora del resumen ejecutivo para una siguiente versión.

7. Pendientes

- Recepción de información desde el área comercial.
- Cierre de la primera versión del informe.

8. Responsables

- Pedro: insistir con el área comercial.
- María: continuar con el diseño del reporte.
- Equipo de análisis: cerrar la primera versión del informe.

9. Próximos pasos

- Pedro realizará seguimiento con comercial.
- El equipo revisará si los datos llegan a tiempo.
- Se preparará la versión para revisión de gerencia el viernes.

10. Riesgos o alertas

- Si la información comercial no llega a tiempo, podría retrasarse la revisión con gerencia.

## Consejo práctico

Para obtener una mejor minuta, no necesitas escribir perfecto durante la reunión.

Puedes tomar notas rápidas y después pedirle a ChatGPT que las ordene. Lo importante es incluir datos clave como:

- tema
- participantes
- acuerdos
- responsables
- fechas
- pendientes

Mientras más explícitos sean esos datos, más útil será la minuta.

## Cuidado con

Las minutas pueden generar malentendidos si incluyen acuerdos que no fueron realmente definidos.

Antes de enviar la minuta, revisa:

- [ ] ¿Los acuerdos realmente se dijeron?
- [ ] ¿Los responsables están correctamente asignados?
- [ ] ¿Las fechas son reales y no inferidas?
- [ ] ¿Los pendientes están claros?
- [ ] ¿No se agregó información inventada?
- [ ] ¿El resumen representa bien la reunión?

## Variante útil

Puedes pedir una versión ejecutiva para enviar por correo:

Convierte esta minuta en un correo breve para enviar al equipo.

Debe incluir:

1. Resumen de la reunión
2. Acuerdos principales
3. Pendientes y responsables
4. Próximos pasos

Minuta:
[pega aquí la minuta]

# Plantilla 5: Extraer compromisos y responsables

## Para qué sirve

Sirve para analizar una reunión, correo, conversación o documento y extraer los compromisos concretos que quedaron definidos.

Esta plantilla ayuda a separar comentarios generales de acciones reales, identificando qué se debe hacer, quién es responsable y cuándo debería completarse.

La idea no es que ChatGPT invente tareas, sino que detecte compromisos explícitos o posibles acciones que aparecen en el texto.

---

## Cuándo usarla

Úsala cuando necesitas ordenar compromisos después de:

- una reunión
- una minuta
- un correo largo
- una conversación con cliente
- una revisión de proyecto
- una sesión de seguimiento
- una planificación interna

Ejemplos típicos:

- identificar responsables después de una reunión
- transformar una minuta en lista de tareas
- preparar seguimiento para el equipo
- revisar qué quedó pendiente con un cliente
- extraer acciones desde una transcripción
- ordenar compromisos antes de enviar un resumen

---

## Prompt listo para copiar

```text
Actúa como un asistente de gestión de tareas y seguimiento.

Necesito analizar el siguiente texto y extraer todos los compromisos, tareas y próximos pasos accionables.

Texto:
[pega aquí el correo, minuta, transcripción o notas]

Contexto:
[explica brevemente de qué trata el texto]

Instrucciones:
- No inventes compromisos.
- Si un responsable no está indicado, escribe "No especificado".
- Si una fecha no está indicada, escribe "No especificada".
- Diferencia entre compromisos explícitos y posibles acciones sugeridas.
- Usa lenguaje claro y concreto.

Devuélveme una tabla con estas columnas:

1. Tipo
2. Compromiso o tarea
3. Responsable
4. Fecha límite
5. Prioridad
6. Estado
7. Observaciones

Después de la tabla, agrega:
- Resumen de los principales pendientes.
- Riesgos si no se hace seguimiento.
- Preguntas que habría que aclarar.
```

## Ejemplo de entrada

Texto:
En la reunión se revisó el avance del proyecto. María indicó que enviará la versión actualizada del documento el miércoles. Pedro quedó de validar los datos con el área comercial. Juan comentó que sería bueno revisar el diseño del reporte, pero no se definió responsable. También se acordó que el equipo tendrá una nueva reunión el viernes para revisar el avance. La fecha de entrega final sigue siendo el lunes.

Contexto:
Reunión interna de seguimiento de proyecto.

## Ejemplo de salida esperada

| Tipo                    | Compromiso o tarea                           | Responsable     | Fecha límite    | Prioridad | Estado      | Observaciones                                          |
| ----------------------- | -------------------------------------------- | --------------- | --------------- | --------- | ----------- | ------------------------------------------------------ |
| Compromiso explícito    | Enviar la versión actualizada del documento  | María           | Miércoles       | Alta      | Pendiente   | Necesario para avanzar con la revisión                 |
| Compromiso explícito    | Validar los datos con el área comercial      | Pedro           | No especificada | Alta      | Pendiente   | Puede afectar la entrega final                         |
| Compromiso explícito    | Realizar nueva reunión de revisión de avance | Equipo          | Viernes         | Media     | Pendiente   | Instancia de seguimiento                               |
| Posible acción sugerida | Revisar el diseño del reporte                | No especificado | No especificada | Media     | Por definir | Se mencionó como recomendación, pero no quedó asignada |

Resumen de principales pendientes:

- María debe enviar el documento actualizado.
- Pedro debe validar los datos con el área comercial.
- El equipo debe reunirse el viernes para revisar avances.
- Falta definir si se revisará el diseño del reporte y quién será responsable.

Riesgos si no se hace seguimiento:

- La validación de datos podría retrasar la entrega final.
- El diseño del reporte podría quedar sin revisión.
- La reunión del viernes podría no tener insumos suficientes.

Preguntas que habría que aclarar:

- ¿Qué día exacto corresponde al miércoles?
- ¿Cuándo debe completar Pedro la validación?
- ¿Quién tomará la revisión del diseño del reporte?

## Consejo práctico

Después de cada reunión importante, puedes usar esta plantilla antes de enviar la minuta.

Esto te ayuda a detectar si realmente quedaron compromisos claros o si solo hubo conversación general.

Una buena instrucción adicional es:

- Marca como "No especificado" cualquier responsable, fecha o acción que no aparezca de forma clara en el texto.

Eso evita que ChatGPT complete datos por su cuenta.

## Cuidado con

No todo comentario es un compromiso.

Por ejemplo:

- Sería bueno revisar el informe.

No es lo mismo que:

- María revisará el informe antes del viernes.

Antes de enviar la lista de tareas, revisa:

- [ ] ¿Los compromisos aparecen realmente en el texto?
- [ ] ¿Los responsables fueron mencionados explícitamente?
- [ ] ¿Las fechas son reales?
- [ ] ¿Se diferenciaron tareas confirmadas de sugerencias?
- [ ] ¿No se agregaron tareas inventadas?
- [ ] ¿La prioridad tiene sentido?

## Variante útil

Puedes pedir una versión lista para seguimiento semanal:

Convierte estos compromisos en una lista de seguimiento semanal.

Formato:

- Tarea
- Responsable
- Fecha compromiso
- Estado actual
- Próxima acción
- Riesgo asociado

Texto:
[pega aquí el texto]

# Plantilla 10: Crear estructura para una presentación

## Para qué sirve

Sirve para transformar una idea, tema o conjunto de información en una estructura clara para una presentación.

Esta plantilla ayuda a ordenar el mensaje, definir una narrativa y proponer una secuencia de diapositivas con objetivo, contenido y recomendación visual.

La idea no es que ChatGPT diseñe la presentación completa por ti, sino que te ayude a construir una base clara antes de llevarla a PowerPoint, Canva o Google Slides.

---

## Cuándo usarla

Úsala cuando necesitas preparar:

- una presentación para jefatura
- una presentación comercial
- una capacitación
- una exposición interna
- una presentación de avance
- una propuesta de proyecto
- un resumen ejecutivo
- una presentación para cliente

Ejemplos típicos:

- presentar resultados de un informe
- explicar una propuesta de mejora
- preparar una capacitación interna
- vender una idea a un equipo
- mostrar avances de un proyecto
- ordenar información antes de diseñar slides

---

## Prompt listo para copiar

```text
Actúa como un especialista en comunicación y estructura de presentaciones.

Necesito crear la estructura de una presentación clara, ordenada y profesional.

Tema de la presentación:
[escribe aquí el tema]

Contexto:
[explica brevemente por qué se hará esta presentación]

Audiencia:
[jefatura / cliente / equipo interno / comité / usuarios / estudiantes / otro]

Objetivo de la presentación:
[informar / convencer / capacitar / vender / reportar avance / solicitar aprobación / otro]

Duración estimada:
[indica duración aproximada]

Nivel de detalle:
[breve / medio / detallado]

Información disponible:
[pega aquí ideas, datos, notas o contenido base]

Tono deseado:
[ejecutivo / comercial / pedagógico / técnico / cercano / formal]

Instrucciones:
- Ordena la presentación con una narrativa clara.
- No inventes datos.
- Si falta información importante, indícalo.
- Sugiere títulos claros para cada diapositiva.
- Incluye una recomendación visual por diapositiva.

Devuélveme:
1. Título sugerido de la presentación.
2. Mensaje principal.
3. Estructura slide por slide.
4. Objetivo de cada slide.
5. Contenido sugerido para cada slide.
6. Recomendación visual para cada slide.
7. Cierre o llamado a la acción.
```

## Ejemplo de entrada

Tema de la presentación:
Propuesta para mejorar el seguimiento de solicitudes internas.

Contexto:
Actualmente las solicitudes se gestionan por correo y no existe trazabilidad clara. Queremos proponer un flujo más ordenado.

Audiencia:
Jefatura y equipo interno.

Objetivo de la presentación:
Solicitar aprobación para iniciar una primera fase de mejora.

Duración estimada:
10 minutos.

Nivel de detalle:
Medio.

Información disponible:

- Hoy las solicitudes llegan por correo.
- No hay estado visible.
- Es difícil saber quién tiene la responsabilidad.
- Se generan retrasos.
- Se propone un formulario centralizado.
- Primera fase: compras menores.
- Se necesitan responsables por etapa.
- Se espera mejorar trazabilidad y tiempos de respuesta.

Tono deseado:
Ejecutivo, claro y profesional.

## Ejemplo de salida esperada

1. Título sugerido

Propuesta de mejora para el seguimiento de solicitudes internas

2. Mensaje principal

Implementar un flujo centralizado permitirá mejorar la trazabilidad, reducir retrasos y ordenar responsabilidades en la gestión de solicitudes internas.

3. Estructura slide por slide

Slide 1: Contexto actual
Objetivo:
Explicar cómo se gestionan hoy las solicitudes.

Contenido sugerido:

- Las solicitudes se reciben principalmente por correo.
- No existe un estado visible del avance.
- La responsabilidad puede quedar poco clara.

Recomendación visual:
Usar un diagrama simple del flujo actual con correo → revisión manual → respuesta.

---

Slide 2: Problemas detectados
Objetivo:
Mostrar por qué el proceso actual genera dificultades.

Contenido sugerido:

- Falta de trazabilidad.
- Dificultad para identificar responsables.
- Retrasos por seguimiento manual.
- Duplicidad o pérdida de información.

Recomendación visual:
Usar una lista de problemas con íconos simples o una matriz de dolor/impacto.

---

Slide 3: Propuesta de solución
Objetivo:
Presentar la idea de un flujo centralizado.

Contenido sugerido:

- Crear un formulario único de ingreso.
- Definir responsables por etapa.
- Generar estados visibles de avance.
- Incorporar notificaciones.

Recomendación visual:
Mostrar un flujo propuesto con etapas: ingreso → revisión → aprobación → cierre.

---

Slide 4: Alcance de la primera fase
Objetivo:
Limitar el alcance inicial para facilitar la aprobación.

Contenido sugerido:

- Enfocar la primera fase en compras menores.
- Validar el flujo con un proceso acotado.
- Ajustar antes de escalar a otras áreas.

Recomendación visual:
Usar una tabla simple con “Incluye / No incluye por ahora”.

---

Slide 5: Beneficios esperados
Objetivo:
Mostrar el valor de implementar la mejora.

Contenido sugerido:

- Mayor trazabilidad.
- Menos seguimiento manual.
- Responsables más claros.
- Mejor tiempo de respuesta.
- Información centralizada.

Recomendación visual:
Usar tarjetas de beneficios o una comparación antes/después.

---

Slide 6: Decisiones requeridas
Objetivo:
Dejar claro qué se necesita aprobar o definir.

Contenido sugerido:

- Validar inicio de primera fase.
- Confirmar responsables.
- Definir indicadores de éxito.
- Acordar fecha de inicio.

Recomendación visual:
Usar una lista tipo checklist.

---

Slide 7: Cierre y próximos pasos
Objetivo:
Cerrar con una solicitud concreta.

Contenido sugerido:

- Confirmar aprobación para iniciar la fase 1.
- Agendar reunión de definición.
- Levantar responsables y reglas.
- Preparar propuesta de implementación.

Recomendación visual:
Usar una línea de tiempo simple con próximos pasos.

4. Cierre o llamado a la acción

Se solicita aprobación para iniciar una primera fase enfocada en compras menores, definir responsables por etapa y preparar el flujo inicial de trabajo.

## Consejo práctico

Antes de diseñar una presentación, primero define el mensaje principal.

Una presentación no debería ser solo una colección de slides. Debería responder una pregunta central:

- ¿Qué quiero que la audiencia entienda, decida o haga al final?

Puedes agregar esta instrucción al prompt:

- Antes de proponer las slides, dime cuál debería ser el mensaje principal de la presentación y qué decisión o acción debería buscar al final.

Esto ayuda a que la presentación tenga dirección.

## Cuidado con

ChatGPT puede proponer demasiadas slides si no le das límites.

Antes de usar la estructura, revisa:

- [ ] ¿La cantidad de slides calza con el tiempo disponible?
- [ ] ¿Cada slide tiene un objetivo claro?
- [ ] ¿No hay slides repetidas?
- [ ] ¿La presentación tiene una narrativa lógica?
- [ ] ¿El cierre pide una acción concreta?
- [ ] ¿No se inventaron datos o beneficios?
